\chapter{Materials and Technologies}

This chapter introduces the core technologies we used to build our super-fast SDF calculator. We'll talk about CUDA (our parallel computing tool), NVIDIA CUB (our high-performance sorting library), NVIDIA OptiX (our ray tracing engine), and the 3D models we used for testing.

\section{CUDA}

Think of your computer's main processor (CPU) like a brilliant professor. They can solve very complex problems, but they can only do one or two at a time. Now, think of a Graphics Processing Unit (GPU) like an army of thousands of elementary school students. Individually, they aren't as smart as the professor, but if you give them thousands of simple math problems, they will finish them all instantly because they work together at the exact same time!

CUDA \cite{cuda} is the special programming interface created by NVIDIA that lets us boss around this army of students. To maximize efficiency, our pipeline heavily utilizes \textbf{CUDA streams}. A stream is essentially an independent queue of commands for the GPU. By default, commands are executed sequentially in a single main stream. However, by creating multiple custom streams, we can launch independent tasks---such as computing vertex normals and building the OptiX BVH---at the exact same time. The GPU executes these streams concurrently, overlapping the workload and significantly reducing the overall execution time of our computation pipeline.

\subsection{NVIDIA CUB}

If CUDA is the language we use to boss the students around, NVIDIA CUB (CUDA Unbound) \cite{cub} is like giving them a highly organized playbook for team exercises. When thousands of threads need to cooperate on a single massive task---like sorting a giant list of edges or finding unique elements---writing that coordination code from scratch is incredibly difficult and prone to errors. CUB provides a set of highly optimized, ready-to-use algorithms that allow our GPU threads to work together flawlessly. We rely on CUB's device-level algorithms (specifically \texttt{DeviceRadixSort} and \texttt{DeviceSelect::Unique}) during our post-processing stage to rapidly sort and deduplicate millions of triangle edges, enabling us to build our adjacency graph in record time.

\subsection{NVIDIA OptiX}

But what happens when we need to shoot rays to calculate thickness? Enter NVIDIA OptiX \cite{optix}. OptiX is a specialized tool designed specifically for ray tracing. Modern NVIDIA graphics cards have dedicated hardware built into them called "RT Cores." These cores are physically designed to do one thing exceptionally well: calculate where a ray hits a triangle. OptiX is the secret key that unlocks this hardware!

OptiX is amazing because it automatically organizes our 3D model into a smart structure called a Bounding Volume Hierarchy (BVH) so that the rays don't have to check every single triangle in the scene. However, setting up OptiX requires a very strict sequence of steps. First, we write our shader code and compile it into a "module." Next, we group those shaders into tasks (like "what to do when a ray hits" vs "what to do when it misses"). We then link everything together into a pipeline, set up our Shader Binding Table (which we'll discuss as our "address book" later!), build the BVH acceleration structure, and finally, launch the rays!
